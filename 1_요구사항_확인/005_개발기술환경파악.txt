개발 기술 환경 파악
- 개요 : 운영체제(OS), 데이터베이스 관리 시스템(DBMS), 미들웨어 고려사항 기술, 오픈소스 주의사항 제시

운영체제(OS; Operating System) 
- 시스템 자원을 효율적으로 관리, 사용자가 편리하게 사용 가능하도록 환경 제공
- 사용자와 하드웨어간 인터페이스 - 시스템 소프트웨어 
- 다른 응용 프로그램이 유용한 작업 가능하도록 환경 제공
- 요구사항 식별 시 고려사항
=> 가용성 (현재 작업 진행 가능여부 파악), 성능, 기술지원(개발에 필요한 매뉴얼 제공여부), 주변 기기, 구축 비용
   
데이터베이스 관리 시스템(DBMS; Database Management System)
- 사용자와 데이터베이스 사이에서 정보 생성, 데이터베이스 관리해주는 소프트웨어
- 데이터의 종속성, 중복성 문제 해결 위함
- 모든 응용 프로그램들이 데이터베이스 공용
- 요구사항 식별 시 고려사항
=> 가용성, 성능, 기술 지원, 상호 호환성(서버와 클라이언트 사이의 호환, 서버 이중화 가능여부), 구축 비용

웹 애플리케이션 서버(WAS; Web Application Server)
- 실시간으로 변하는 동적인 콘텐츠 처리하는 미들웨어(운영체제가 제공하는 서비스 이외에 추가적인 서비스 제공)
- 라이브러리 제공
- 데이터베이스 서버와 연동해 사용
- 고려사항
=> 가용성, 성능, 기술 지원, 구축 비용

오픈 소스(Open Source)
- 제한없이 누구나 사용 가능하도록 소스 코드 공개
- 고려사항
=> 라이선스의 종류, 사용자 수, 기술의 지속 가능성(계속 업데이트 되는지 여부)
