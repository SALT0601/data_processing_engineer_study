클래스(Class) 다이어그램

정적 모델링
- 사용자 요구 기능을 구현하는데 필요한 자료의 논리적 구조를 개발자 관점에서 그림으로 표현
   (기능 모델링은 이와 달리 사용자 관점에서 기능 작동에 대해 설명)
- 객체들 사이에 어떤 관련이 있는지 구조적 관점(View)에서 표현
- 객체들을 클래스로 추상화하여 표현 => 현실 세계의 객체들이 가질 수 있는 속성과 동작으로 추상적으로 표현한 것.
- UML 정적 모델링의 대표

클래스 다이어그램
- 클래스와 클래스가 가지는 속성, 클래스 사이의 관계
- 시스템 구성 요소에 대해 이해 가능한 구조적 다이어그램
- 시스템 구성 요소 문서화하는데 사용

구성 요소
- 클래스 - p.74참고 - 객체의 속성과 오퍼레이션(동작) 표현, 
                         - 보통 3개의 구획(Compartment)로 나눠 이름, 속성, 오퍼레이션 표기
                         - 속성(Attribute) : 클래스 상태나 정보 / 오퍼레이션(Operation) = 함수(메소드, Method) : 클래스 수행 동작
- 제약조건 - 메모지 이미지 - 속성에 입력될 값에 대한 제약조건, 오퍼레이션 수행 전후에 지정해야 할 조건, 클래스 안에 기술할 경우 중괄호{ }
- 관계(Relationships) - 클래스와 클래스 사이의 연관성, 연관관계/집합관계/포함관계/일반화관계/의존관계

연관 클래스
- 연관 관계에 있는 두 클래스에 추가 표현할 속성이나 오퍼레이션 있는 경우 생성
- 연관 관계 선 가운데로부터 점선으로 연관 클래스 이어 표시
- 이름은 연관 관계의 이름 이용해 지정
