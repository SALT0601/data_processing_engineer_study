요구사항 분석
- 소프트웨어 개발의 첫 단계, 요구사항을 이해하고 문서화(명세화)하는 활동
- 타당성 조사, 비용, 일정 제약 설정
- 사용자의 요구를 정확하게 추출해야해서 설계보다 중요한 단계

구조적 분석 기법
- 자료의 흐름과 처리를 중심으로 하는 요구사항 분석 방법
- 입력, 출력, 인쇄 기능 즉 모듈별로 기능을 따로 만들어 연결해서 프로그램을 만드는것
- 도형 중심의 도구와 절차를 이용해 문서화함
- 하향식 방법을 사용하여 시스템 세분화(ex. 기업의 조직도 처럼)
- 분석의 중복 배제 가능
- 주요 구조적 분석 기법 도구
=> 자료 흐름도(DFD), 자료 사전(DD), 소단위 명세서(Mini-Spec.), 개체 관계도(ERD), 상태 전이도(STD), 제어 명세서

자료 흐름도(DFD; Data Flow Diagram) = 자료 흐름 그래프, 버블 차트
- 자료의 흐름 및 변환 과정과 기능을 도형중심으로 기술

자료 흐름도 기본 기호
- 프로세스(Process) :  처리과정으로 처리, 기능, 변환, 버블이라고 함 - 동그라미 기호
- 자료 흐름(Data Flow) : 자료의 이동, 연관관계 - 화살표 기호
- 자료 저장소(Data Store) : 데이터베이스 - 직선 기호 2개로 가운데에 설명 들어감
- 단말(Treminator) : 입출력 개체 - 사각형 기호

자료 사전(DD; Data Dictionary) = 데이터의 데이터, 메타 데이터(Meta Data)
- 자료흐름도에 있는 자료를 더 자세히 정의하고 기록
- 데이터를 설명하는 데이터
- 표기 기호
=> = : 자료의 정의
     + : 자료의 연결
     ( ) : 자료의 생략
     [ ] : 자료의 선택
     { } : 자료의 반복
     * * : 자료의 설명, 주석
