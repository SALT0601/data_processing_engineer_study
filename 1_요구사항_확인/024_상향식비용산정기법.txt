비용 산정 기법 - 상향식

상향식 비용 산정 기법
- 서부적인 작업 단위별로 비용 산정 후 집계하여 전체 비용 산정
- 주요 기법 - LOC(원시 코드 라인 수) 기법 / 개발 단계별 인월수 기법 / 수학적 산정 기법

LOC(source Line Of Code) 기법
- 각 기능의 원시 코드 라인 수의 비관치(가장 많이 측정된 코드 라인 수), 낙관치(가장 적게 측정된 코드 라인 수),
  기대치(측정된 모든 코드 라인 수의 평균)를 측정하여 예측치 구하고 이를 이용해 비용 산정
- 측정이 용이하고 이해가 쉬워 가장 많이 사용
- 예측치를 이용해 생산성, 노력, 개발 기간 등 비용 산정
- 예측치 = a+4m+b/6 단, a:낙관치, b:비관치, m:기대치(중간치)
- 산정공식 => 공식 암기하기
=> - 노력(인월) = 개발 기간 * 투입 인원
                    = LOC(예측치) / 1인당 월평균 생산 코드 라인 수
     - 개발 비용 = 노력(인월) * 단위 비용(1인당 월평균 인건비)
     - 개발 기간 = 노력(인월) / 투입 인원
     - 생산성 = LOC / 노력(인월)

개발 단계별 인월수(Effort Per Task) 기법
- LOC기법 보완을 위한 기법, 각 기능을 구현시키는 데 필요한 노력을 생명 주기의 각 단계별로 산정
- LOC보다 더 정확
