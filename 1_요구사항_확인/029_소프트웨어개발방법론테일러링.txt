소프트웨어 개발 방법론 테일러링
- 소프트웨어 개발 방법론의 절차, 사용기법 등을 수정 및 보완하는 작업
- 실제 업무에서 여건에 맞게 수정, 보완 -> 맞춤 양복처럼
- 절차 
=> 프로젝트 특징 정의 -> 표준 프로세스 선정 및 검증 -> 상위 수준의 커스터마이징 -> 세부 커스터마이징 -> 테일러링 문서화

소프트웨어 개발 방법론 테일러링 고려사항
내부적 기준
=> 목표 환경 : 개발 환경과 유형이 서로 다른 경우
     요구사항 : 우선적 고려할 요구사항이 서로 다른경우
     프로젝트 규모 : 규모가 서로 다른 경우
     보유 기술 : 프로세스, 구성원 능력 등이 서로 다른 경우
외부적 기준
=> 법적 제약사항 : IT Compliance(반드시 지켜야 할 법적 규제 사항이나 지침)가 서로 다른 경우
     표준 품질 기준 : 금융, 제도 등 분야별 표준 품질 기준이 서로 다른 경우
