애플리케이션 테스트
- 애플리케이션에 잠재되어있는 결함 찾아내는 일련의 행위 또는 절차
- 개발된 소프트웨어가 고객의 요구사항 만족시키는지 확인하고 소프트웨어가 기능 정확히 수행하는지 검증

애플리케이션 테스트의 기본 원리
- 완벽한 테스트 불가능 : 소프트웨어의 잠재적 결함을 줄일 수 있지만 결함이 없다고 증명할 수는 없음
- 파레토 법칙 : 애플리케이션의 20%에 해당하는 코드에서 전체 결함의 80%가 발견됨
- 상출제 패러독시 : 동일한 테스트 케이스로 동일한 테스트를 반복하면 더 이상 결함이 발견되지 않는 현상
- 테스팅은 정황 의존 : 소프트웨어 특징, 테스트 환경, 테스터의 역량 등 정황에 따라 테스트 결과 달라질 수 있으므로 정황에 따라 테스트 다르게 수행
- 오류-부재의 게변 : 소프트웨어의 결함을 모두 제거해도 사용자의 요구사항 만족시키지 못하면 해당 소프트웨어는 품질이 높다고 말할 수 없음
- 테스트와 위험은 반비례 : 테스트를 많이 하면 할수록 미래에 발생할 위험 줄임
- 테스트의 점진적 확대 : 테스트는 작은 부분에서 시작해 점점 확대하며 진행해야 함
- 테스트의 별도 팀 수행 : 테스트는 개발자와 관계없는 별도의 팀에서 수행
