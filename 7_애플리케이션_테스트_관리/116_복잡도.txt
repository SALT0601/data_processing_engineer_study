복잡도
- 시스템이나 시스템 구성요소 또는 소프트웨어의 복잡한 정도
- 어느 정도 수준까지 테스트해야 하는지 또는 개발하는데 어느 정도 자원 소요되는지 예측

시간 복잡도
- 알고리즘 수행하기 위해 프로세스가 수행하는 연산 횟수 수치화한 것
- 낮을수록 알고리즘 실행시간 짧고, 높을수록 긺
- 점근 표기법 종류
-> 빅오 표기법 : 알고리즘 실행시간이 최악일 때 표기
                   : 입력값에 대해 알고리즘 수행했을 때 명령어의 실행 횟수는 어떠한 경우에도 표기 수치보다 많을 수 없음
-> 세타 표기법 : 알고리즘 실행시간 평균, 명령어의 실행 횟수의 평균적인 수치
-> 오메가 표기법 : 실행시간 최상일때, 명령어의 실행 횟수는 어떠한 경우에도 표기 수치보다 적을 수 없음

빅오 표기법으로 표현한 최악의 알고리즘 시간 복잡도
- O(1) : 입력값(n)에 관계 없이 일정하게 문제 해결에 하나의 단계만 거침 - 스택의 삽입, 삭제
- O(log2n) : 문제 해결에 필요한 단계가 입력값 또는 조건에 의해 감소 - 이진트리, 이진검색
- O(n) : 단계가 입력값과 일대일 관계 - for문
- O(nlog2n) : 단계가 n(log2n)번만큼 수행 - 힙 정렬, 2-way 합병 정렬
- O(n^2) : 단계가 입력값의 제곱만큼 수행 - 삽입 정렬, 쉘 정렬, 선태 정렬, 버블 정렬, 퀵 정렬
- O(2^n) : 단계가 2의 입력값 제곱만큼 수행 - 피보나치 수열

순환 복잡도 = 멕케이브 순환도 = 멕케이브 복잡도 메트릭
- 논리적 복잡도 측정하기 위한 소프트웨어의 척도
- 제어 흐름도 이론에 기초를 둠
- 제어 흐름도 G에서 순환 복잡고 V(G)는 2가지 방법으로 계산 가능 
-> 1. 순환 복잡도는 제어 흐름도의 영역 수와 일치해 영역 수 계산
    2. V(G) = E - N + 2 : E는 화살표 수, N은 노드의 수
