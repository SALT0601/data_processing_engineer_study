인터넷
- 전 세계 수많은 컴퓨터와 네트워크들이 연결된 광범위한 컴퓨터 통신망
- 인터넷에 연결된 모든 컴퓨터는 고유한 IP 주소를 갖음

IP 주소
- 인터넷에 연결된 모든 컴퓨터 자원을 구분하기 위한 고유한 주소
- 8비트씩 4부분, 총 32비트로 구성
- 네트워크 부분의 길이에 따라 A~E 클래스까지 총 5단계로 구성
- A Class : 국가나 대형 통신망(0~127로 시작 / 0과 127은 예약어라 실질적으로 1~126 씀)
	: 2^24 = 19,777,216개의 호스트 사용 가능 -> 네트워크 1 덩어리 가짐
- B Class : 중대형 통신망(128~191로 시작)
	: 2^16 = 65,536개의 호스트 사용 가능 -> 네트워크 2 덩어리 가짐
- C Class : 소규모 통신망(192~223로 시작)
	: 2^8 = 256개의 호스트 사용 가능 -> 네트워크 3 덩어리 가짐 
- D Class : 멀티캐스트(화상회의 등) 용(224~239로 시작)
- E Class : 실험적 주소이며 공용되지 않음 

서브네팅
- 네트워크 주소를 여러개의 작은 네트워크로 나누어 사용하는 것
- 4바이트의 IP주소 중 네트워크 주소와 호스트 주소를 구분하기 위한 비트를 서브넷 마스크라 하며 이를 변경해 네트워크 주소 여러개로 분할 사용
- 서브넷 마스크는 각 클래스마다 다르게 사용

IPv6(Internet Protocol version 6)
- 현재 사용하고 있는 IP주소 체계인 IPv4의 주소 부족 문제를 해결하기 위해 개발
- 128비트의 긴 주소를 사용해 주소 부족 문제 해결
- IPv4에 비해 자료 전송 속도가 빠름
- 인증성, 기밀성, 데이터 무결성의 지원으로 보안 문제 해결
- 주소의 확장성, 융통성, 연동성이 뛰어남
- 실시간 흐름 제어로 향상된 멀티미디어 기능 지원
- 패킷크기를 확장할 수 있어 패킷 크기에 제한이 없다
- 기본 헤더 뒤에 확장 헤더를 더해 더욱 다양한 정보의 저장이 가능해 네트워크 기능 확장이 용이
- IPv4와 호환성이 뛰어남

IPv6의 구성
- 16비트씩 8부분, 총 128비트
- 각 부분을 16진수로 표현, 콜론(:)으로 구분
- 유니캐스트 : 단일 송신자와 단일 수신자 간 통신(일대일 통신)
- 멀티캐스트 : 단일 송신자와 다중 수신자 간 통신(일대다 통신)
- 애니캐스트 : 단일 송신자와 가장 가까이 있는 단일 수신자 간 통신(일대일 통신)

도메인 네임
- 숫자로 된 IP주로를 문자 형태로 표현
- 호스트 컴퓨터 이름, 소속 기관 이름, 소속기관 종류, 소속 국가명 순으로 구성, 왼쪽에서 오른쪽으로 갈수록 상위 도메인
-> ex. www.info.co.kr
- 문자로 된 도메인 네임을 IP주소로 변환하는 역할을 DNS라 하며 이 서버를 DNS 서버라 함
