Secure OS
- 보안 기능을 갖춘 커널을 이식해 외부의 침입으로부터 시스템 자원을 보호하는 운영체제
- 보안 커널은 보안 기능을 갖춘 커널로 TCB를 기반으로 참조 모니터의 개념 구현하고 집행
- 보호 방법(구현하기 복잡한것부터)
-> 암호적 분리 : 내부 정보 암호화
-> 논리적 분리 : 프로세스의 논리적 구역 지정해 구역을 벗어나는 행위 제한
-> 시간적 분리 : 동일 시간에 하나의 프로세스만 수행되도록 해 동시 실행으로 발생하는 보안 취약점 제거
-> 물리적 분리 : 사용자별 특정 장비만 사용하도록 제한
- 보안 기능 : 식별 및 인증, 임의적/강제적 접근통제, 객체 재사용 보호, 완전한 조정, 신뢰 경로, 감사 및 감사기록 축소 등

참조 모니터
- 보호 대상 객체에 대한 접근통제를 수행하는 추상머신 -> 이를 실제로 구현한 것이 보안 커널
- 보안 커널 데이터베이스를 참조해 객체에 대한 접근 허가 여부 결정
- 참조 모니터와 보안 커널의 특징
-> 격리성 : 부정 조적이 불가능해야 함
-> 검증가능성 : 적절히 구현되었다는 것을 확인할 수 있어야 함
-> 완전성 : 우회가 불가능해야 함
