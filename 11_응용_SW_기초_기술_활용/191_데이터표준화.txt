데이터 표준화
- 시스템을 구성하는 데이터 요소의 명칭, 정의, 형식, 규칙에 대한 원칙 수립하고 적용하는 것
- 데이터 표준화 작업을 통해 사용자가 데이터를 정확히 이해하고 활용할 수 있게 표준 항목명 부여

데이터 표준화 구성 요소
- 데이터 표준 : 데이터 모델이나 DB에서 정의할 수 있는 모든 오브젝트를 대상으로 데이터 표준화 수행
	      : 종류
                    -> 표준 단어 : 업무에서 사용하고 있는 일정한 의미를 가진 최소 단위의 단어
                    -> 표준 도메인 : 문자형, 숫자형, 날짜형, 시간형과 같이 컬럼을 성질에 따라 묶은 것
                    -> 표준 코드 : 선택할 수 있는 값을 정형화하기 위해 기준에 맞게 정의된 코드 값, 도메인의 한 유형
                    -> 표준 용어 : 단어, 도메인, 코드 표준이 정의되면 이를 바탕으로 표준 용어 구성
- 데이터 관리 조직 : 데이터 표준 원칙이나 데이터 표준의 준수 여부 등 관리하는 사람들, 데이터 관리자 등
- 데이터 표준화 절차 : 데이터 표준화 요구사항 수집 -> 데이터 표준 정의 -> 데이터 표준 확정 -> 데이터 표준 관리 순으로 진행

데이터 표준화의 대상
- 데이터 명칭 : 유일성, 의미 전달 충분성, 업무적 보편성 갖는 이름으로 정의
- 데이터 정의 : 3자도 쉽게 이해하도록 해당 데이터가 의미하는 범위 및 자격 요건 등 규정
- 데이터 형식 : 일관되게 데이터 형식 정의 -> 오류나 통제 위험 최소화
- 데이터 규칙 : 데이터 값을 사전에 지정해 데이터 정합성 및 완전성 향상
