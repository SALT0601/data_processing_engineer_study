페이지 교체 알고리즘
- 페이지 부재가 발생하면 어떤 페이지 프레임을 선택하여 교체할 것인지를 결정하는 기법

OPT(OPTimal replacement, 최적 교체)
- 앞으로 가장 오랫동안 사용하지 않을 페이지 교체
- 벨레이디가 제안
- 페이지 부재 횟수가 가장 적게 발생하는 가장 효율적인 알고리즘

FIFO
- 가장 먼저 들어와서 가장 오래 있었던 페이지 교체
- 이해하기 쉽고 프로그래밍 및 설계 간단

LRU(Least Recently Used)
- 최근에 가장 오랫동안 사용하지 않은 페이지 교체
- 각 페이지마다 계수기(페이지가 사용될때마다 0으로 클리어)나 스택을 두어 현시점에서 가장 오랫동안 사용하지 않은 가장 오래전에 사용된 페이지 교체

LFU(Least Frequently Used)
- 사용 빈도가 가장 적은 페이지 교체
- 활발하게 사용되는 페이지는 사용 횟수가 많아 교체되지 않고 사용

NUR(Not Used Recently)
- LRU와 비슷, 최근에 사용하지 않은 페이지 교체
- 후에도 사용되지 않을 가능성 높다는 것을 전제로 LRU에서 나타나는 시간적인 오버헤드 줄일 수 있음
- 최근 사용 여부 확인인을 위해 각 페이지마다 두 개의 비트, 즉 참조비트(페이지 호출되면 1, 아니면 0)와 변형 비트(페이지 사용될 때 1, 아닐때 0)가 사용
- 참조비트가 0이고 변형 비트가 1일때 참조비트가 1이고 변형 비트가 0일때 보다 먼저 교체

SCR(Second Cjance Replacement, 2차 기회 교체)
- 가장 오랫동안 주기억장치에 있던 페이지 중 자주 사용되는 페이지의 교체 방지 위함
- FIFO 단점 보완
