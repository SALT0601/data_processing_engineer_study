입력 데이터 검증 및 표현

소프트웨어 개발의 구현단계에서 검증해야하는 보안 점검내용 7가지
- 입력 데이터 검증 및 표현 / 보안 기능 / 시간 및 상태 / 에러처리 / 코드 오류 / 캡슐화 / API 오용

입력 데이터 검증 및 표현
- 입력 데이터로 발생하는 문제들을 예방하기 위해 구현 단계에서 검증해야하는 보안 점검 항목들
- 입력 데이터 검증 및 표현 약점
-> SQL 삽입 : 웹 응용 프로그램에 SQL 삽입해 내부 DB서버의 데이터를 유출 및 변조하고 관리자 인증을 우회하는 보안 약점
-> 경로 조작 및 자원 삽입 : 데이터 입출력 경로를 조작해 서버 자원을 수정 삭제할 수 있는 보안 약점
-> 크로스사이트 스크립팅 : 웹페이지에 악의적인 스크립트 삽입해 방문자들의 정보를 탈취, 비정상적인 기능 수행 유발
-> 운영체제 명령어 삽입 : 외부 입력값을 통해 시스템 명령어의 실행 유도해 권한 탈취, 시스템 장애 유발
-> 위험한 형식 파일 업로드 : 악의적인 명령어가 포한된 스크립트 파일을 업로드해 시스템에 손상, 시스템을 제어할 수 있음
-> 신뢰되지 않은 URL 주소로 자동접속 연결 : 입력값으로 사이트 주소 받는 경우 이를 조작해 방문자를 피싱 사이트로 유도
-> 메모리 버퍼 오버플로 : 연속된 메모리 공간을 사용하는 프로그램에서 할당된 메모리의 범위를 넘어선 위치에서 자료를 읽거나 쓰려고 할 때 발생
