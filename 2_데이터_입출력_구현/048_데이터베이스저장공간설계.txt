데이터베이스 저장 공간 설계
- 테이블이나 컬럼 등 실제 데이터가 저장되는 공간 정의
- 테이블
-> 테이터베이스의 가장 기본 객체
    로우와 컬럼으로 구성
    모든 데이터는 테이블에 저장
- 컬럼
-> 열을 구성하는 요소
    데이터 타입, 길이 등으로 정의
- 테이블스페이스
-> 테이블이 저장되는 논리적 영역
    한 개의 테이블스페이스에 한 개 이상의 테이블 저장 가능

테이블 종류
- 일반 테이블 = 힙 구조 테이블 
-> 데이터 정렬하지 않고 가장 적절한 기억 장소에 저장 후 임의의 방식으로 데이터 관리하는 힙구조를 사용
    대부분의 DBMS에서 표준 테이블로 사용
- 클러스터드 인덱스 테이블
-> 기본키나 인덱스키의 순서에 따라 데이터 저장
    일반적인 인덱스를 사용하는 테이블에 비해 접근 경로 단축
- 파티셔닝 테이블 : 대용량의 테이블을 작은 논리적 단위인 파티션으로 나눔
- 외부 테이블
-> 데이터베이스에서 일반 테이블처럼 이용할 수 있는 외부 파일
    데이터베이스 내 객체로 존재
- 임시 테이블
-> 트랜잭션이나 세션별로 데이터 저장하고 처리
    임시 테이블에 저장된 데이터는 트랜잭션이 종료되면 삭제됨
