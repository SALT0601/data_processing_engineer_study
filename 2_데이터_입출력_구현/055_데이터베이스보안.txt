데이터베이스 보안
- 데이터베이스의 일부, 전체에 대해 권한이 없는 사용자가 액세스하는 것 금지위해 사용
- 데이터 단위는 테이블 전체로부터 특정 테이블의 특정행과 열에 있는 데이터 값에 이르까지 다양

암호화(Encryption)
- 데이터 보낼 때 송신자가 지정한 수신자 이외에 그 내용 알 수 없도록 평문을 암호문으로 변환
- 암호화 과정 : 암호화되지 않은 평문을 암호문으로 바꿈
- 복호화(Decryption) : 암호문을 원래의 평문으로 바꿈
- 암호화 기법 : 개인키 암호 방식, 공개키 암호 방식

접근통제
- 데이터가 저장된 객체(테이블, 컬럼 등 DB 개체)와 이를 사용하려는 주체사이의 정보 흐름 제한 -> 사용자의 접근 통제해 데이터 보호
- 접근통제 3요소 : 접근통제 정책, 접근통제 메커니즘, 접근통제 보안모델
- 접근통제 기술
-> 임의 접근통제(DAC, Discretionary Access Control)
     - 데이터에 접근하는 사용자 신원에 따라 접근 권한 부여
     - 데이터 소유자가 접근통제 권한 지정하고 제어
     - 객체를 생성한 사용자가 생성된 객체에 대한 모든 권한 부여받고 부여된 권한 다른 사용자에게 허가 가능
-> 강제 접근통제(MAC, Mandatory Access Control)
     - 주체와 객체의 등급 비교해 접근 권한 부여
     - 시스템이 접근통제 권한 지정
     - 데이터베이스 객체별 보안 등급 부여 가능
     - 사용자별로 인가 등급 부여 가능
-> 역할기반 접근통제(RBAC, Role Based Access Control)
     - 사용자 역할에 따라 접근 권한 부여
     - 중앙관리자가 접근통제 권한 지정
     - 임의 접근통제와 강제 접근통제의 단점 보완
     - 다중 프로그래밍 환경에 최적화

접근통제 정책
- 어떤 주체가 언제, 어디서, 어떤 객체에게, 어떤 행위에 대한 허용 여부 정의
- 종류
-> 신분 기반 정책 
     - 주체나 그룹의 신분에 근거해 객체 접근 제한(IBP, GBP)
     - IBP(Individual-Based Policy) : 최소 권한 정책, 단일 주체에게 하나의 객체에 대한 허가 부여
     - GBP(Group-Based Policy) : 복수 주체에 하나의 객체에 대한 허가부여
-> 규칙 기반 정책
     - 주체가 갖는 권한에 근거해 객체의 접근 제한(MLP,CBP)
     - MLP(Multi-Level Policy) : 사용자나 객체별로 지정된 기밀 분류에 따른 정책
     - CBP(Compartment-Based Policy) : 집단별로 지정된 기밀 허가에 따른 정책   
-> 역할 기반 정책 : GBP의 변형된 정책, 주체의 신분이 아닌 주체가 맡은 역할에 근거해 객체 접근 제한

접근통제 매커니즘
- 정의된 접근통제 정책을 구현하는 기술적 방법
- 접근통제 목록(객체 기준, 어떤 주체가 어떤 행위 할 수 있는지 기록), 능력 리스트(주체 기준, 주체에게 허가된 자원 및 권한 기록), 보안등급, 패스워드, 암호화 등

접근통제 보안 모델
- 보안 정책을 구현하기 위한 정형화된 모델
- 종류
-> 기밀성 모델
     - 군사적 목적으로 개발된 최초의 수학적 모델
     - 기밀성 보장이 최우선
     - 군대 시스템 등 특수 환경에서 주로 사용
-> 무결성 모델 : 기밀성 모델에서 발생하는 불법적 정보 변경을 방지하기 위해 무결성 기반으로 개발된 모델
-> 접근통제 모델
     - 접근통제 매커니즘을 보안 모델로 발전
     - 대표적으로 접근통제 행렬(Access Control Matrix)
     - 접근통제 행렬 : 임의적 접근통제 관리하기 위한 보안보델, 행은 주체, 열은 객체, 행과 열로 주체와 객체의 권한 유형 나타냄

접근통제 조건
- 접근통제 매커니즘의 취약점 보완위해 접근통제 정책에 부가해 적용 가능한 조건
- 값 종속 통제(Value-Dependent Control) : 보통 객체에 저장된 값에 상관 없이 접근통제 동일하게 허용, 값 종속 통제는 객체에 저장된 값에 따라 다르게 접근 통제 허용해야하는 경우 사용
- 다중 사용자 통제(Multi-User Control) : 지정된 객체에 다수의 사용자가 동시에 접근 요구하는 경우
- 컨텍스트 기반 통제(Context-Based Control) : 특정시간, 네트워크 주소 등에 근거해 접근 제어, 다른 보안 정책과 결합해 보안 시스템의 취약점 보완할 때 사용

감사 추적
- 사용자나 애플리케이션이 데이터베이스에 접근해 수행한 모든 활동 기록
- 오류가 발생한 데이터베이스 복구, 부적절한 데이터 조작 파악
