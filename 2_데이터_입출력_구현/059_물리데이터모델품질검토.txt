물리 데이터 모델 품질 검토
- 물리 데이터 모델을 설계하고 데이터베이스 객체 생성 후 개발 단계로 넘어가기 전 모델러와 이해관계자 모여 수행
- 목적은 데이터베이스 성능 향상광 오류 예방
- 검토하려면 모든 이해관계자가 동의하는 검토 기준이 필요

물리 데이터 모델 품질 기준
- 정확성 : 데이터 모델이 요구사항이나 업무 규칙, 표기법에 따라 정확하게 표현
- 완전성 : 데이터 모델의 구성 요소 누락 없이 정의, 요구사항이나 업무 영역 누락없이 반영
- 준거성 : 데이터 표준, 표준화 규칙, 법적 요건 등 전확하게 준수
- 최신성 : 최근의 이유나 현행 시스템 반영
- 일관성 : 표현상의 일관성 유지
- 활용성 : 작성된 모델과 설명을 사용자가 충분히 이해 가능하고 업무 변화에 따른 데이터 구조의 변경 최소화 가능하도록 설계

물리 데이터 모델 검토 항목
- 물리데이터 모델의 특성을 반영한 품질 기준 작성 후 이를 기반으로 작성
- 테이블, 컬럼, 무결성 제약 조건 등 주요 구성 요소와 반정규화, 인덱스, 스토리지 등 전반적인것 검토 항목으로 작성
