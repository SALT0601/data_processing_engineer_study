오류 데이터 측정 및 정제
- 고품질 데이터 운영 및 관리 위해 수행
- 진행 순서(과정)
=> 데이터 품질 분석 (오류 데이터 찾기 위해 원천 및 목적 시스템 데이터 정합성(데이터 값이 서로 모순 없이 일관되게 일치하는 정도) 여부 확인)
-> 오류 데이터 측정 (정상 데이터와 오류 데이터의 수 측정해 오류 관리 목록 작성)
-> 오류 데이터 정제 (오류 관리 목록의 항목 분석해 원천 데이터 정체 혹은 전환 프로그램 수정)

오류 상태
- Open : 오류 보고만 되고 분석 안됨
- Assigned : 개발자에게 오류 전달
- Fixed : 개발자가 오류 수정
- Closed : 수정된 오류 테스트해서 오류 발견 안됨
- Deferred : 오류 수정 연기
- Clarified/Classified : 보고된 오류 관련자들이 확인했을 때 오류 아니라고 확인

데이터 정제요청서
- 데이터 정제와 관련된 전반적인 내용 문서로 작성
- 오류 관리 목록을 기반으로 데이터 정제 요건 목록 작성하고 목록의 항목별로 데이터 정제요청서 작성

데이터 정제 보고서
- 정제된 원천 데이터가 정상적으로 정제되었는지 확인한 결과를 문서로 작성
- 정제 요청 데이터와 정제된 데이터 항목 눈으로 비교하여 확인
