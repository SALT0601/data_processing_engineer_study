관계형 데이터베이스의 제약 조건 - 무결성

무결성(Integrity)
- 데이터베이스에 저장된 데이터 값과 현실 세계의 실제값이 일치하는 정확성
- 저장된 데이터베이스에 잘못된 데이터가 없다는 것
- 무결성 제약조건 : 정확성을 위해 부정확한 자료가 데이터베이스에 저장되는 것 방지 위한 제약조건

무결성의 종류
- 개체 무결성 : 기본키의 어떤 속성도 Null 값, 중복값 가질 수 없다는 규정
- 참조 무결성 : 외래키 값은 Null, 참조 릴레이션의 기본키 값과 동일 -> 참조할 수 없는 외래키 값 못 가짐
- 도메인 무결성 : 주어진 속성 값이 정의된 도메인에 속한 값
- 사용자 정의 무결성 : 속성 값들이 사용자가 정의한 제약조건에 만족
- NULL 무결성 : 특정 속성 값이 NULL 될 수 없도록
- 고유 무결성 : 특정 속성에 대해 각 튜플이 갖는 속성값들이 서로 달라야 함
- 키 무결성 : 하나의 릴레이션에는 적어도 하나의 키 존재
- 관계 무결성 : 릴레이션에 어느 한 튜플의 삽입 가능 여부, 한 릴레이션과 다른 릴레이션의 튜플들 사이의 관계에 대한 적절성 여부

데이터 무결성 강화
- 데이터 품질에 직접적 영향을 미치므로 적절한 무결성 정의하고 강화해야 함
- 애플리케이션(데이터 생성, 수정, 삭제 시 무결성 조건을 검증하는 코드를 프로그램 내에 추가),
  데이터베이스 트리거(트리거(이벤트가 발생할 때마다 자동으로 수행되는 절차형 SQL) 이벤트에 무결성 조건 실행하는 절차형 SQL 추가),
  제약 조건(제약 조건 성정해 무결성 유지)을 이용해 강화 가능
