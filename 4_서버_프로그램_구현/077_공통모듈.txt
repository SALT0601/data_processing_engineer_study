공통 모듈
- 여러 프로그램에서 공통으로 사용할 수 있는 모듈
- 자주 사용되는 계산식이나 매번 필요한 사용자 인증과 같은 기능들이 공통 모듈로 구성
- 공통 모듈을 구현할 때는 해당 기능 명확히 이해할 수 있도록 명세 기법 준수

공통 모듈 명세 기법의 종류
- 정확성(Correctness) : 시스템 구현 시 해당 기능이 필요하다는 것 알 수 있게 정확히 작성 
- 명확성(Clarity) : 해당 기능 이해할 때 중의적으로 해석되지 않도록 명확히 작성  
- 완전성(Completeness) : 시스템 구현을 위해 필요한 모든 것 기술 
- 일관성(Consistency) : 공통 기능들 간 상호 충돌 발생하지 않도록 작성 
- 추적성(Traceability) : 기능에 대한 요구사항의 출처, 관련 시스템 등 관계를 파악할 수 있도록 작성

재사용(Reuse)
- 이미 개발된 기능들을 새로운 시스템이나 기능 개발에 사용하기 적합하도록 최적화
- 새로 개발하는데 필요한 비용과 시간 절약
- 누구나 이해할 수 있고 사용 가능하도록 사용법 공개
- 재사용 규모에 따른 분류
-> 함수와 객체 : 클래스나 메소드 단위의 소스 코드를 재사용
    컴포넌트 : 컴포넌트 자체에 대한 수정 없이 인터페이스를 통해 통신하는 방식으로 재사용
    애플리케이션 : 공통된 기능들 제공하는 애플리케이션을 고유하는 방식으로 재사용

효과적인 모듈 설계 방안 - 가볍게 읽어보고 넘어가기
- 결합도 줄이고 응집도 높여 모듈의 독립성과 재사용성 높임
- 복잡도와 중복성 줄이고 일관성 유지
- 모듈의 기능은 예측 가능해야하고 지나치게 제한적이면 안됨
- 모듈 크기는 시스템의 전반적 기능과 구조를 이해하기 쉬운 크기로 분해
- 효과적 제어를 위해 모듈 간 계층적 관계 정의하는 자료 제시
