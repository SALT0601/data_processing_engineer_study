수학적 산정 기법 = 경험적 추정 모형, 실험적 추정 모형
- 상향식 비용 산정 기법
- 개발 비용 산정의 자동화를 목표
- 공식은 과거 유사한 프로젝틀 기반으로 유도된 것
- 주요 기법 => COCOMO 모형(LOC이용), Putnam 모형(LOC이용), 기능 점수(FP이용) 모형

COCOMO(COnstructive COst MOdel) 모형
- 원시프로그램의 규모인 LOC에 의한 기법
- LOC 예측 후 소프트웨어 종류에 따라 자르게 책정되는 비용 산정 방정식에 대입하여 비용 산정
- 결과는 프로젝트 완성하는 데 필요한 노력(Man-Month)로 나타냄
- 보헴이 제안

COCOMO의 소프트웨어 개발 유형 -> LOC 규모에 따라 나눠짐(KDSI : 전체 라인 수를 1000라인 단위로 묶은것)
- 조직형(Organic Mode) : 5만(50KDSI) 라인 이하 개발, 중,소규모의 소프트웨어, 사무처리용, 업무용, 과학용 등에 적합
- 반분리형(Semi-Detached Mode) : 30만(300KDSI) 라인 이하 개발, 중간형 소프트웨어, 트랜젝션 처리, 운영체제, DBMS와 컴파일러 같은 유틸리티 개발에 적합
- 내장형(Embedded Mode) : 30만(300KDSI) 라인 이상 개발, 초대형 규모, 트랜젝션 처리, 운영체제와 신호기 제어 시스템, 실시간 처리 시스템 등 시스템 프로그램 개발에 적합

COCOMO 모형의 종류
- 기본형 : 소프트웨어 크기(LOC)와 개발 유형만 이용해 비용 산정
- 중간형 : 기본형의 공식을 토대로 사용 + 4가지 특성(제품, 컴퓨터, 개발 요원, 프로젝트)에 의해 비용 산정
- 발전형 : 중간형 보완 모형, 자세하고 정확하게 노력을 산출하여 비용 산정, 환경과 구성 요소가 사전에 정의, 개발 과정의 후반부에 주로 적용

Putnam 모형 = 생명 주기 예측 모형
- 소프트웨어 생명 주기의 전 과정 동안 사용될 노력의 분포를 예상
- 푸트남이 제안
- 시간에 따른 함수로 표현되는 Rayleigh-Norden 곡선(노든이 경험적 자료 수집한 근거로 그린 곡선)의 노력 분포도를 기초로 함
- 대형 프로젝트의 노력 분포 산정에 이용
- 개발 기간이 늘어날수록 프로젝트 적용 인원의 노력이 감소함

기능 점수(FP; Function Point) 모형
- 기능 증대시키는 요인별 가중치 부여, 가중치 합산하여 총 기능 점수 산출, 총 점수와 영향도 이용해 기능 점수(FP) 구한 후 비용 산정
- 알브레히트가 제안
- 소프트웨어 기능 증대 요인
=> 자료 입력(입력 양식), 정보 출력(출력 보고서), 명령어(사용자 질의수), 데이터 파일, 필요한 외부 루틴과의 인터페이스

비용 산정 자동화 추정 도구
- SLIM : Rayleigh-Norden 곡선, Putnam 예측 모델을 기초로 개발
- ESTIMACS : FP모형 기초로 개발 -> 다양한 프로젝트, 개인별 요소 수용 위함
